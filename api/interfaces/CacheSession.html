Interface CacheSession

A session for performing cache requests.

Applications must use {@link Session#createCacheSession} to construct this class.

The supplied CacheSessionProperties will be copied. Subsequent modifications to the passed properties will not modify the session. The properties may be reused.

sendCacheRequest

  • sendCacheRequest(requestID: number, topic: Destination, subscribe: boolean, liveDataAction: CacheLiveDataAction, cbInfo: CacheCBInfo): void
    • Defined in lib/solace.model.ts:3705

    Issues an asynchronous cache request. The result of the request will be returned via the listener.

    throws

    {OperationError} In the following cases:

    • If the CacheSession is disposed. Subcode: ErrorSubcode.INVALID_OPERATION
    • If one or more parameters were invalid. Subcode: ErrorSubcode.PARAMETER_INVALID_TYPE
    • If the supplied topic and live data action cannot be combined. Subcode: ErrorSubcode.PARAMETER_CONFLICT
    • If the supplied topic or live data action cannot be used given the current outstanding requests. Subcode: ErrorSubcode.PARAMETER_CONFLICT

    Parameters

    • requestID: number

      The application-assigned ID number for the request.

    • topic: Destination

      The topic destination for which the cache request will be made.

    • subscribe: boolean

      If true, the session will subscribe to the given {Topic}, if it is not already subscribed, before performing the cache request.

    • liveDataAction: CacheLiveDataAction

      The action to perform when the CacheSession receives live data on the given topic.

    • cbInfo: CacheCBInfo

      Callback info for the cache request.

    Returns void
